## 📝 Test Cases

### 1. บันทึกมิเตอร์ย้อนหลัง (Backdated Meter Reading)

#### Test Case 1.1: บันทึกย้อนหลังเกินกำหนด
**ขั้นตอน:**
1. Login เป็น EMPLOYEE
2. ไปที่เมนู "บันทึกค่าน้ำ"
3. เลือกบ้านที่ต้องการบันทึก
4. กรอกวันที่: 3 เดือนที่แล้ว (เช่น 2025-11-26)
5. กรอกค่ามิเตอร์: 1500
6. กด "บันทึก"

**ผลลัพธ์ที่คาดหวัง:**
- ❌ ระบบแจ้งเตือน: "ไม่สามารถบันทึกค่าย้อนหลังเกิน 30 วันได้"
- ❌ หรือต้องการการอนุมัติจาก ADMIN
- ❌ ไม่มีการบันทึกข้อมูลเข้า database

#### Test Case 1.2: บันทึกย้อนหลังในกำหนด
**ขั้นตอน:**
1. Login เป็น EMPLOYEE
2. ไปที่เมนู "บันทึกค่าน้ำ"
3. เลือกบ้านที่ต้องการบันทึก
4. กรอกวันที่: 5 วันที่แล้ว
5. กรอกค่ามิเตอร์: 1500
6. กด "บันทึก"

**ผลลัพธ์ที่คาดหวัง:**
- ✅ บันทึกสำเร็จ
- ✅ ข้อมูลบันทึกเข้า database

---

### 2. ทดสอบ Billing ซ้ำ (Duplicate Billing)

#### Test Case 2.1: สร้างบิลซ้ำจากการอ่านเดิม
**ขั้นตอน:**
1. Login เป็น EMPLOYEE หรือ ADMIN
2. ไปที่เมนู "บันทึกค่าน้ำ"
3. บันทึกค่ามิเตอร์บ้าน A
4. ไปที่เมนู "ใบแจ้งหนี้"
5. กด "สร้างใบแจ้งหนี้" จากการอ่านครั้งที่ 1
6. บันทึกสำเร็จ
7. พยายามกด "สร้างใบแจ้งหนี้" อีกครั้งจากการอ่านเดียวกัน

**ผลลัพธ์ที่คาดหวัง:**
- ❌ ระบบแจ้งเตือน: "มีใบแจ้งหนี้สำหรับการอ่านนี้อยู่แล้ว"
- ❌ ไม่สามารถสร้างบิลซ้ำได้
- ❌ ไม่มีการสร้างบิลใหม่ใน database

#### Test Case 2.2: สร้างบิลจากการอ่านใหม่
**ขั้นตอน:**
1. บันทึกค่ามิเตอร์บ้าน A อีกครั้ง (เดือนถัดไป)
2. ไปที่เมนู "ใบแจ้งหนี้"
3. กด "สร้างใบแจ้งหนี้" จากการอ่านครั้งใหม่

**ผลลัพธ์ที่คาดหวัง:**
- ✅ สร้างบิลสำเร็จ
- ✅ มีบิลใหม่ในระบบ

---

### 3. ทดสอบพร้อมกันหลายคน (Concurrent Users)

#### Test Case 3.1: หลายคน login พร้อมกัน
**ขั้นตอน:**
1. เปิด Browser 3 อัน (Chrome, Edge, Firefox) หรือ 3 Incognito windows
2. ในแต่ละ browser login ด้วย account ต่างกัน:
   - Browser 1: user1@example.com
   - Browser 2: user2@example.com
   - Browser 3: user3@example.com
3. พร้อมกัน กด "เข้าสู่ระบบ" ในแต่ละ browser

**ผลลัพธ์ที่คาดหวัง:**
- ✅ ทั้ง 3 คน login สำเร็จ
- ✅ แต่ละคนได้ token ที่แตกต่างกัน
- ✅ Session แยกกันอย่างถูกต้อง

#### Test Case 3.2: หลายคนบันทึกค่าน้ำพร้อมกัน
**ขั้นตอน:**
1. เปิด Browser 2 อัน login เป็น EMPLOYEE
2. Browser 1: เตรียมบันทึกบ้าน A
3. Browser 2: เตรียมบันทึกบ้าน B
4. กด "บันทึก" พร้อมกัน

**ผลลัพธ์ที่คาดหวัง:**
- ✅ ทั้งสองการบันทึกสำเร็จ
- ✅ ไม่มี deadlock หรือ race condition
- ✅ ข้อมูลถูกต้อง

#### Test Case 3.3: Rate Limiting - พยายาม login ผิดพลาดหลายครั้ง
**ขั้นตอน:**
1. ไปที่หน้า login
2. กรอกรหัสผ่านผิด 10 ครั้งติดต่อกัน
3. ครั้งที่ 11 กรอกรหัสถูกต้อง

**ผลลัพธ์ที่คาดหวัง:**
- ❌ หลังจาก 5 ครั้ง ระบบแจ้ง: "กรุณารอสักครู่ก่อนลองใหม่"
- ❌ หรือ: "บัญชีถูกล็อกชั่วคราว 15 นาที"
- ❌ ครั้งที่ 11 (รหัสถูก) ก็ยังไม่ให้เข้าจนกว่าจะครบเวลา

### 4. ทดสอบ Token หมดอายุ (Expired Token)

#### Test Case 4.1: Token หมดอายุขณะใช้งาน
**ขั้นตอน:**
1. Login เข้าระบบ
2. รอให้ token หมดอายุ (หรือแก้ไข localStorage ให้ token เป็นอันเก่า)
3. พยายามทำ action ใดก็ได้ (เช่น บันทึกค่าน้ำ)

**ผลลัพธ์ที่คาดหวัง:**
- 🔄 ระบบพยายาม refresh token อัตโนมัติ
- ✅ ถ้า refresh สำเร็จ: ดำเนินการต่อ
- ❌ ถ้า refresh ไม่สำเร็จ: redirect ไปหน้า login พร้อมแจ้ง "เซสชันหมดอายุ"

#### Test Case 4.2: Refresh token หมดอายุด้วย
**ขั้นตอน:**
1. Login เข้าระบบ
2. รอให้ทั้ง access token และ refresh token หมดอายุ
3. หรือลบ cookie refresh token จาก browser
4. พยายามทำ action

**ผลลัพธ์ที่คาดหวัง:**
- ❌ Redirect ไปหน้า login
- ❌ แจ้งเตือน: "กรุณาเข้าสู่ระบบใหม่"
- ✅ ไม่สามารถทำงานใดๆ ได้จนกว่าจะ login ใหม่

### 5. ทดสอบ Upload ไฟล์ผิดประเภท

#### Test Case 5.1: Upload ไฟล์ .txt
**ขั้นตอน:**
1. Login เข้าระบบ
2. ไปที่หน้าโปรไฟล์ผู้ใช้
3. กด "เปลี่ยนรูปโปรไฟล์"
4. เลือกไฟล์: `test.txt` (สร้างไฟล์ text ธรรมดา)
5. กด "อัปโหลด"

**ผลลัพธ์ที่คาดหวัง:**
- ❌ ระบบแจ้ง: "รองรับเฉพาะไฟล์รูปภาพ (JPG, PNG, WEBP) เท่านั้น"
- ❌ ไม่มีการอัปโหลดไป Cloudinary
- ✅ รูปโปรไฟล์ไม่เปลี่ยน

#### Test Case 5.2: Upload ไฟล์ .exe (executable)
**ขั้นตอน:**
1. สร้างไฟล์ `malware.exe` (หรือเปลี่ยนชื่อไฟล์ .txt เป็น .exe)
2. พยายามอัปโหลดเป็นรูปโปรไฟล์

**ผลลัพธ์ที่คาดหวัง:**
- ❌ ระบบแจ้ง: "ไฟล์ไม่ปลอดภัย หรือไม่รองรับประเภทไฟล์นี้"
- ❌ Cloudinary ปฏิเสธไฟล์

#### Test Case 5.3: Upload ไฟล์ขนาดใหญ่เกินไป (>10MB)
**ขั้นตอน:**
1. สร้างไฟล์รูปภาพขนาดใหญ่ (>10MB)
2. พยายามอัปโหลด

**ผลลัพธ์ที่คาดหวัง:**
- ❌ ระบบแจ้ง: "ไฟล์ขนาดใหญ่เกินไป (สูงสุด 5MB)"
- ❌ ไม่มีการอัปโหลด

#### Test Case 5.4: Upload ไฟล์รูปภาพที่ถูกต้อง
**ขั้นตอน:**
1. ใช้ไฟล์ `.jpg` หรือ `.png` ขนาดปกติ (<5MB)
2. อัปโหลดรูปโปรไฟล์

**ผลลัพธ์ที่คาดหวัง:**
- ✅ อัปโหลดสำเร็จ
- ✅ รูปแสดงในโปรไฟล์
- ✅ ไฟล์อยู่บน Cloudinary

### 6. ทดสอบสิทธิ์ Role (Role-Based Access Control)

#### Test Case 6.1: USER พยายามเข้า Admin Panel
**ขั้นตอน:**
1. สร้าง user ใหม่ด้วย role = USER
2. Login ด้วย user นั้น
3. พยายามเข้า URL: `/admin`
4. หรือพยายามกดเมนู admin (ถ้ามี)

**ผลลัพธ์ที่คาดหวัง:**
- ❌ Redirect ไปหน้า `/billing` หรือ `/unauthorized`
- ❌ แจ้งเตือน: "คุณไม่มีสิทธิ์เข้าถึงหน้านี้"
- ❌ API calls ไปยัง endpoints สำหรับ admin ตอบกลับด้วย 403 Forbidden

#### Test Case 6.2: EMPLOYEE พยายามจัดการ Users
**ขั้นตอน:**
1. Login เป็น EMPLOYEE
2. พยายามไปที่ `/admin/users`
3. พยายามกด "เพิ่มผู้ใช้ใหม่"

**ผลลัพธ์ที่คาดหวัง:**
- ❌ ไม่สามารถเข้าหน้าจัดการ users ได้
- ❌ ไม่มีปุ่ม/เมนูสำหรับจัดการ users
- ❌ API POST /users ตอบกลับ 403

#### Test Case 6.3: EMPLOYEE เข้าถึงหน้าบันทึกค่าน้ำ
**ขั้นตอน:**
1. Login เป็น EMPLOYEE
2. ไปที่ `/admin/readings`
3. บันทึกค่าน้ำ

**ผลลัพธ์ที่คาดหวัง:**
- ✅ เข้าหน้าได้
- ✅ บันทึกค่าน้ำได้
- ✅ เห็นเฉพาะบ้านใน village ที่ตัวเองดูแล

#### Test Case 6.4: ADMIN จัดการทุกอย่างได้
**ขั้นตอน:**
1. Login เป็น ADMIN
2. เข้า `/admin` - ควรเข้าได้
3. จัดการ users - ควรได้
4. จัดการ houses - ควรได้
5. จัดการ billing - ควรได้
6. ดู dashboard - ควรได้

**ผลลัพธ์ที่คาดหวัง:**
- ✅ เข้าถึงทุกหน้าได้
- ✅ ทำทุก operation ได้
- ✅ เห็นข้อมูลทุก village

#### Test Case 6.5: SUPER_ADMIN ลบ Village
**ขั้นตอน:**
1. Login เป็น SUPER_ADMIN
2. ไปที่จัดการ Village
3. พยายามลบ Village (ที่มีข้อมูล)

**ผลลัพธ์ที่คาดหวัง:**
- ⚠️ แสดง warning: "การลบ Village จะลบข้อมูลทั้งหมดที่เกี่ยวข้อง"
- ✅ ถ้ายืนยัน ให้ลบได้
- ❌ ADMIN ธรรมดาไม่เห็นปุ่ม/เมนูลบ village

#### Test Case 6.6: USER เห็นเฉพาะบิลตัวเอง
**ขั้นตอน:**
1. Login เป็น USER (เจ้าของบ้าน)
2. ไปที่หน้า "ใบแจ้งหนี้ของฉัน"

**ผลลัพธ์ที่คาดหวัง:**
- ✅ เห็นเฉพาะบิลของบ้านตัวเอง
- ❌ ไม่เห็นบิลของบ้านอื่น
- ❌ ไม่เห็นข้อมูลส่วนตัวของ user อื่น
